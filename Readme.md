# RAFS - Roman Altshuler File System

_это дедушкина фамилия, авторство принадлежит Захарченко_

## В чём суть

Суть заключалась в том, чтобы реализовать файловую систему, которая хранит все данные исключительно в оперативной памяти. Основной концептуальный подход был такой: нужно создать набор функций для работы с файлами/директориями и присвоить их базовым структурам ядра Linux.

### Архитектурный подход

В Linux все файловые системы работают через Virtual File System что является унифицированным интерфейсом. Чтобы создать свою ФС, нам нужно:

1. Определить структуры данных - создать rafs_file для хранения файлов в RAM и rafs_sb_info для информации о ФС
2. Написать функции-обработчики - реализовать все необходимые операции (создание, чтение, запись и т.д.)
3. Присвоить функции структурам - создать file_operations, inode_operations и связать их с нашими функциями
4. Интегрировать в VFS - зарегистрировать ФС в ядре через file_system_type

По сути, я "наклепал" кучу функций и присвоил их в базовые структуры - это и есть основная идея реализации своей файловой системы.

## Архитектура и компоненты

### Файлы проекта

- rafs.c - главный модуль ядра - точка входа и выхода
- super.c - управление суперблоком - монтирование/размонтирование ФС
- inode.c - операции с inode - создание/удаление файлов и директорий
- file.c - чтение/запись файлов + управление памятью
- dir.c - просмотр содержимого директорий

### Дополнителььные структуры данных

```c
struct rafs_file {                    // Структура для файла в RAM
    struct list_head list;            // Связывание в список
    char name[256];                   // Имя файла
    ino_t ino;                        // Уникальный номер inode
    ino_t parent_ino;                 // Родительская директория
    umode_t mode;                     // Права + тип 
    char *data;                       // Указатель на данные в RAM
    size_t size;                      // Размер файла
    size_t capacity;                  // Выделенная память
    unsigned int link_count;          // Счетчик жестких ссылок
};

struct rafs_sb_info {                 // Информация о суперблоке
    struct list_head file_list;       // Список всех файлов ФС
    struct rw_semaphore rwsem;        // Семафор для синхронизации
    ino_t next_ino;                   // Следующий свободный номер ноды
};
```

## Описание основных функций

### 1. Инициализация модуля (rafs_init/rafs_exit)

Просто регистрируем/(отменяем регистрацию) файловую систему в списке доступных файловых систем ядра. 

### 2. Монтирование (rafs_mount → rafs_fill_super)

При вызове mount -t rafs ядро вызывает функцию rafs_mount которая позволяет монтировать виртуальные файловые системы, т.е. системы не привязанные ни к какому физическому устройству и существующие исключительно в RAM

Функция rafs_fill_super подготавливает файловую систему к работе добавляя ей корень. (инициализирует всё, создаёт корневую директорию под inode 1000 и т.д.)


### 3. Создание файла (rafs_create)

При вызове touch или создании файла через другие команды ядро вызывает rafs_create которая создаёт новый файл в нашей RAM файловой системе.

Функция проверяет не существует ли уже файл с таким именем, создаёт структуру rafs_file, выделяет память и регистрирует файл в VFS (создаёт inode и dentry).

### 4. Чтение файла (rafs_read)

При вызове cat или других команд чтения ядро вызывает rafs_read которая копирует данные из RAM структуры в буфер пользовательского приложения.

Функция извлекает данные из rafs_file структуры и через copy_to_user передаёт их в пользовательское пространство обновляя позицию чтения.

### 5. Запись файла (rafs_write)

При вызове echo или других команд записи ядро вызывает rafs_write которая копирует данные от пользователя в нашу RAM структуру.

Функция через copy_from_user получает данные от пользователя, автоматически расширяет буфер файла при необходимости (удваивая размер) и сохраняет данные в RAM.

### 6. Просмотр директории (rafs_iterate)

При вызове ls или других команд просмотра директорий ядро вызывает rafs_iterate которая выводит список файлов в директории.

Функция перебирает все файлы принадлежащие данной директории и через dir_emit передаёт информацию о каждом файле (имя, тип, inode номер) для отображения.

### 7. Создание директории (rafs_mkdir)

При вызове mkdir ядро вызывает rafs_mkdir которая создаёт новую директорию в нашей RAM файловой системе.

Функция создаёт структуру rafs_file для директории, выделяет память, регистрирует в VFS и связывает с родительской директорией.

### 8. Удаление директории (rafs_rmdir)

При вызове rmdir ядро вызывает rafs_rmdir которая удаляет директорию из нашей RAM файловой системы.

Функция проверяет что директория пуста, удаляет её из списка файлов и освобождает память только если это была последняя ссылка на директорию.

### 9. Создание жестких ссылок (rafs_link)

При вызове ln (без опции -s) ядро вызывает rafs_link которая создаёт жесткую ссылку на существующий файл.

Функция создаёт новую запись в списке файлов с тем же inode номером, увеличивает счётчик ссылок и регистрирует новую ссылку в VFS.

---
Далее просто подкладываем данные функции в структуры VFS и любуемся как прекрасно они работают.
Это выглядит вот так:
```c
struct file_operations rafs_file_ops = {
    .owner = THIS_MODULE,
    .llseek = generic_file_llseek,
    .read = rafs_read,
    .write = rafs_write,
};

.... OR ....

struct inode_operations rafs_inode_ops = {
.lookup = rafs_lookup,
.create = rafs_create,
.unlink = rafs_unlink,
.mkdir = rafs_mkdir,
.rmdir = rafs_rmdir,
.link = rafs_link,
};
```

И при создании новый inode главное не забывать добавлять эти структуры в их поля вот так:
```c
    inode->i_op = &rafs_inode_ops;
    inode->i_fop = &rafs_dir_ops;
```

## Сборка и использование


```bash
make                                  # сборка модуля
sudo insmod rafs.ko                   # загрузка модуля в ядро
sudo mount -t rafs "TODO" /mnt/rafs   # монтирование модуля

# работа с ФС
cd /mnt/rafs
ls -la
touch file1
echo "test" > file1
...


sudo umount rafs                      # размонтирование модуля
sudo rmmod rafs                   # выгрузка модуля из ядра
```
